## 🌐 Open-Source Tools — Cloud Cost Space

### 🟢 Measure Layer (Visibility & Allocation)

**1. OpenCost** *(Best match to your agent)*
- CNCF project — provides near-real-time cost updates, integrates cleanly into existing engineering workflows, and acts as a reusable cost data layer rather than a standalone reporting tool [Spendbase](https://www.spendbase.co/blog/saas-management/open-source-cloud-cost-optimization-tools/)
- Cloud-agnostic, works on AKS/EKS/GKE
- **Fits your agent as**: data source feeding your LLM analysis layer
- Free, self-hosted

**2. Kubecost** *(K8s specialist)*
- Provides hyper-granular allocation by any Kubernetes concept — namespace, deployment, pod — and attaches in-cluster costs (CPU, RAM) to out-of-cluster expenses like databases and storage [Costimizer](https://costimizer.ai/blogs/cloud-cost-optimization-tools)
- The core is open-source and free to install — zero-cost if you have the expertise to run it [Cloudchipr](https://cloudchipr.com/blog/best-cloud-cost-optimization-tools)
- **Gap vs your agent**: monitoring only, no automation or LLM reasoning

**3. OptScale** *(Most complete open-source)*
- Supports AWS, Azure, GCP, Alibaba Cloud, and Kubernetes — includes cost analytics, anomaly detection, rightsizing recommendations, and budget management. Also covers MLOps use cases including cost optimization for AI workloads [CloudZero](https://www.cloudzero.com/blog/open-source-cloud-cost-management/)
- **Gap vs your agent**: no LLM-based reasoning, rule-based only

**4. Steampipe**
- Lets teams query live cloud APIs using standard SQL — no ETL pipeline or database setup required [CloudZero](https://www.cloudzero.com/blog/open-source-cloud-cost-management/)
- **Fits your agent as**: query engine for Month 3 multi-cloud expansion

**5. Infracost**
- IaC cost estimation — scans Terraform plans before apply
- Useful for both prevention (policy guardrails) and measure layers [Spendbase](https://www.spendbase.co/blog/saas-management/open-source-cloud-cost-optimization-tools/)
- **Fits your agent as**: pre-deployment cost gate in CI/CD pipelines

---

### 🔴 Act Layer (Policy Enforcement & Automation)

**6. Cloud Custodian (c7n)**
- Write YAML policies to filter resources (e.g. unencrypted S3 buckets) and trigger actions (e.g. tag non-compliant EC2 instances or terminate them). Supports AWS, Azure, and GCP with real-time enforcement via serverless functions or cron jobs [Flexera](https://www.flexera.com/blog/finops/finops-tools/)
- **Gap vs your agent**: YAML rules engine, not AI-driven reasoning

**7. Kubernetes VPA (Vertical Pod Autoscaler)**
- Continuously observes CPU and memory usage and provides data-driven target request recommendations — directly attacks one of the most common hidden cost drivers by correcting inflated requests [Spendbase](https://www.spendbase.co/blog/saas-management/open-source-cloud-cost-optimization-tools/)
- **Fits your agent as**: Month 5 auto-remediation action for K8s rightsizing

## 🏆 Strategic Recommendation for Samsung

The smartest move is a **hybrid stack**:

```
OpenCost / Kubecost     → Data collection layer (K8s costs)
Steampipe               → Multi-cloud SQL queries
Cloud Custodian         → Policy enforcement rules
        ↓  Feed data into  ↓
Your CloudCostOps Agent → LLM reasoning + memory + actions
        ↓  Output to  ↓
Grafana + Slack         → Reporting & alerts
```

This way your agent doesn't compete with open-source tools — it **orchestrates them** with AI on top, which is the differentiator no existing tool has. That's your innovation story.
